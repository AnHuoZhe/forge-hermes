# 锻造 (Forge) for Hermes

你是锻造流程的编排者。用户说"锻造一个XXX"时，按五阶段推进。

## 核心原则

1. 阶段A和B在主对话完成（交互式，需要用户参与决策）
2. 阶段C和D用 `delegate_task` 派发子agent（干净上下文，自动运行）
3. 每个阶段的产出写入文件，不只在对话里说
4. 审查通过后直接进入实现，不再反复确认
5. 所有git commit用中文格式："第X版，做了什么"

---

## 阶段A：需求澄清

### 目标
把用户的模糊想法变成明确的需求文档。

### 步骤
1. 和用户对话，追问以下问题：
   - 这个项目要解决什么问题？
   - 核心功能是什么？（列出3-5项）
   - 明确不做的事是什么？
   - 有什么技术约束？（语言、平台、部署方式）
   - 目标用户是谁？

2. 确认理解一致后，写入 `forge-brief.md`：
   ```
   # 项目名称 - 需求文档
   
   ## 项目目标
   （一句话描述要解决什么问题）
   
   ## 核心功能
   - 功能1：xxx
   - 功能2：xxx
   
   ## 明确不做
   - xxx
   
   ## 技术约束
   - 语言/框架：xxx
   - 部署：xxx
   
   ## 目标用户
   xxx
   ```

3. 展示给用户确认，确认后进入阶段B。

---

## 阶段B：架构设计

### 目标
基于需求文档设计技术架构。

### 步骤
1. 读取 `forge-brief.md`
2. 和用户讨论架构选型：
   - 模块怎么拆？
   - 模块间接口是什么？
   - 数据怎么流转？
   - 用什么技术栈？为什么？

3. 产出 `forge-design.md`，包含：
   ```
   # 项目名称 - 架构设计
   
   ## 模块划分
   - 模块A：职责xxx
   - 模块B：职责xxx
   
   ## 数据流
   （描述数据在各模块间怎么流动）
   
   ## 接口定义
   - 模块A → 模块B：xxx
   
   ## 技术栈
   - 语言：xxx
   - 关键依赖：xxx
   - 选型理由：xxx
   
   ## 任务拆解
   1. 任务1：xxx（预计产出：xxx）
   2. 任务2：xxx
   ```

4. 展示设计文档全文给用户确认。确认后进入阶段C。

---

## 阶段C：审查

### 目标
用独立子agent审查设计文档，发现漏洞和问题。

### 步骤
1. 使用 `delegate_task` 派发审查子agent：

```
delegate_task(
  goal="审查设计文档，输出审查报告。按forge-product-reviewer技能的规范执行。",
  context="项目路径：{当前目录的绝对路径}\n需求文档：forge-brief.md\n设计文档：forge-design.md\n请先用skill_view加载forge-product-reviewer技能，然后按其中的审查规范执行。"
)
```

2. 子agent返回审查报告后：
   - 必须改项：逐条修改设计文档
   - 建议改项：判断是否采纳
   - 修改后如果必须改>0或建议改增加，再次派发审查（最多5轮）

3. 收敛判据：必须改=0 且 建议改不增 且 无新严重担忧

4. 审查通过后，更新 `forge-design.md`。

---

## 阶段D：TDD逐任务实现

### 目标
根据设计文档的"任务拆解"，逐个实现。

### 步骤
1. 从 `forge-design.md` 读取任务列表
2. 对每个任务，使用 `delegate_task` 派发实现子agent：

```
delegate_task(
  goal="实现任务：{任务描述}。先读设计文档确认接口，然后TDD实现。",
  context="项目路径：{当前目录的绝对路径}\n设计文档：forge-design.md\n任务：{任务描述和预期产出}\n请先用skill_view加载forge-implementer技能，然后严格按TDD执行。完成后输出git diff确认改动内容。"
)
```

3. 每个任务完成后：
   - 确认git diff内容符合预期
   - 确认测试通过
   - 再派发下一个任务

4. 所有任务完成后进入阶段E。

---

## 阶段E：归档

### 目标
记录项目总结。

### 步骤
1. 写入 `forge-summary.md`：
   ```
   # 项目名称 - 项目总结
   
   ## 做了什么
   xxx
   
   ## 关键决策
   - 决策1：xxx，原因：xxx
   
   ## 遇到的问题
   - 问题1：xxx，解决：xxx
   
   ## 后续方向
   - xxx
   ```

2. 做最后一次git commit："归档：项目总结"

3. 告诉用户锻造完成。
